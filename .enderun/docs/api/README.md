# API Contracts Registry

English | [Türkçe](#türkçe)

This directory is the central hub for all domain-specific API contracts. It ensures that `@backend` and `@frontend` are perfectly synchronized before a single line of application code is written.

## English

### Principles

1. **Contract-First:** No endpoint is implemented until its documentation exists here.
2. **Domain Driven:** Contracts are grouped by domain (e.g., `auth.md`, `users.md`).
3. **SSOT:** This directory is the Single Source of Truth for all HTTP interactions.

### Workflow

1. `@backend` creates a new domain file (e.g., `orders.md`) using the OpenAPI/Markdown standard.
2. `@manager` or `@analyst` audits the contract for standard compliance (Pagination, Error shapes).
3. Once approved, `@backend` implements the logic and `@frontend` builds the UI.

## Türkçe

Bu dizin, projeye özgü tüm API kontratlarının merkezi deposudur. Uygulama kodu yazılmadan önce `@backend` ve `@frontend` ekiplerinin tam bir uyum içinde olmasını sağlar.

### Prensipler

1. **Önce Kontrat:** Dökümantasyonu burada yer almayan hiçbir endpoint kodlanamaz.
2. **Domain Bazlı:** Kontratlar alanlarına göre gruplanır (örn: `auth.md`, `users.md`).
3. **SSOT:** Bu dizin, tüm HTTP etkileşimleri için "Tek Gerçeklik Kaynağı"dır.

### İş Akışı

1. `@backend` yeni bir domain dosyası oluşturur (örn: `orders.md`).
2. `@manager` veya `@analyst` kontratın standartlara (Sayfalama, Hata yapıları) uygunluğunu denetler.
3. Onaylandıktan sonra kodlama başlar.
